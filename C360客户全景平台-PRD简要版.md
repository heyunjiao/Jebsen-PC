# C360 客户全景平台 - 产品需求文档（简要版）

## 1. 项目概述

### 1.1 项目背景

C360 客户全景平台是一个面向 B 端企业的客户数据管理平台，旨在帮助企业实现客户数据的统一管理、智能分群、精准营销和商机转化。

### 1.2 产品定位

- **核心价值**：统一客户身份（OneID）、数据质量保障、智能客户分群、商机管理
- **目标客户**：企业运营人员、数据分析师、营销人员、系统管理员
- **应用场景**：客户数据管理、精准营销、商机挖掘、数据质量治理

### 1.3 技术架构

- **前端框架**：Vue 3.4+ (Composition API)
- **UI 组件库**：Element Plus 2.7+
- **状态管理**：Pinia
- **构建工具**：Vite 5+
- **国际化**：支持简体中文、繁体中文、英文

## 2. 功能模块清单

### 2.1 核心业务模块

#### 2.1.1 欢迎页 (Home)

- **功能描述**：系统首页，展示欢迎信息和快捷入口
- **客户场景**：客户登录后进入系统首页

#### 2.1.2 Dashboard - 数据监控看板

- **功能描述**：数据可视化监控看板
- **子功能**：
  - 数据监控看板：实时数据监控、指标展示
- **客户场景**：运营人员查看系统整体数据概况

#### 2.1.3 数据处理模块

**2.1.3.1 异常中心 (Error Correction)**

- **功能描述**：数据质量异常检测与处理中心
- **核心功能**：
  - 异常任务列表（待处理/已处理）
  - 异常分类：有效性、唯一性、完整性、一致性、合规性
  - 身份冲突解决工作台（合并/拆分）
  - 数据质量快速编辑
  - 同步失败重试
  - 统计面板（异常数量、处理进度）
- **客户场景**：
  - 数据质量专员发现数据异常，进入异常中心处理
  - 系统自动检测到身份冲突，需要人工决策合并或拆分
  - 数据同步失败，需要重试同步

**2.1.3.2 审批中心 (Approval)**

- **功能描述**：数据合并、分群等操作的审批流程
- **核心功能**：
  - 待我审批列表
  - 已处理历史记录
  - 快速审批（批准/驳回）
  - 审批详情查看（风险提示、置信度评估）
- **客户场景**：
  - 管理员收到数据合并申请，需要审批
  - 运营人员提交分群申请，等待审批通过

#### 2.1.4 客户管理模块

**2.1.4.1 客户列表 (Customer List)**

- **功能描述**：客户客户统一管理
- **核心功能**：
  - 客户列表查询（支持多条件筛选）
  - 360度全景视图（OneID、基本信息、行为数据、标签、分群）
  - 客户维护（编辑客户信息）
  - 纠错反馈（提交数据错误反馈）
  - 数据脱敏显示（手机号、证件号）
  - 多值字段展示（多个手机号、地址等）
  - 生命周期状态管理
  - 数据导出
- **客户场景**：
  - 运营人员查看客户详细信息
  - 客服人员维护客户客户
  - 客户发现数据错误，提交纠错反馈

**2.1.4.2 客户批量操作 (Batch Operation)**

- **功能描述**：批量处理客户数据
- **客户场景**：运营人员需要批量更新客户信息或执行批量操作

#### 2.1.5 客户筛选与分群 (Customer Segmentation)

- **功能描述**：可视化客户分群构建工具
- **核心功能**：
  - **漏斗筛选模式**：快速筛选（基础属性、客户偏好、客户行为、客户价值、车辆标签、服务习惯、标签）
  - **自定义筛选模式**：规则构建器（AND/OR 逻辑组合）
  - 冲突检测（标签-分群互斥、数值范围错误）
  - 分群预估（预估人数、覆盖率）
  - 样本预览
  - 洞察面板（分群特征分析）
  - 保存为分群（需审批）
  - 导出分群数据
- **客户场景**：
  - 营销人员需要筛选"高价值客户"进行精准营销
  - 运营人员构建"即将流失客户"分群，进行挽留活动
  - 数据分析师分析客户特征，创建分群

#### 2.1.6 标签管理 (Tag Management)

- **功能描述**：客户标签的创建、管理和自动化规则配置
- **核心功能**：
  - 标签列表（支持分类、状态筛选）
  - 标签创建/编辑（名称、分类、描述、规则配置）
  - 标签类型：自动化规则标签、人工手动标签
  - 标签状态：草稿、已发布、已停用、已废弃
  - 规则编辑器（支持复杂逻辑组合）
  - 覆盖人数统计
  - 审核流程（审核中状态提示）
- **客户场景**：
  - 运营人员创建"高净值客户"标签，配置自动化规则
  - 数据分析师管理标签分类和生命周期

#### 2.1.7 分群管理 (Segment Management)

- **功能描述**：客户分群的统一管理
- **核心功能**：
  - 分群列表（支持状态筛选）
  - 分群创建/编辑（名称、描述、规则配置）
  - 分群状态：草稿、计算中、生效、失败
  - 成员数统计、快照数统计
  - 分群预览（成员列表）
  - 推送商机（将分群推送为商机）
  - 导出分群数据
  - 审核流程（审核中状态提示）
- **客户场景**：
  - 运营人员查看已创建的分群，推送为商机
  - 数据分析师导出分群数据进行分析

#### 2.1.8 商机管理 (Lead Management)

**2.1.8.1 商机看板 (Dashboard)**

- **功能描述**：商机数据可视化看板
- **核心功能**：
  - 价值分层（钻石、黄金、白银等层级统计）
  - 生命周期监控（各生命周期阶段统计）
  - 时间区间筛选
  - 数据刷新
  - 快速跳转到商机列表、规则配置
- **客户场景**：运营人员查看商机整体概况，了解商机分布和趋势

**2.1.8.2 商机列表 (List)**

- **功能描述**：商机列表查询和管理
- **客户场景**：运营人员查看具体商机信息，进行跟进

**2.1.8.3 规则配置 (Rule Config)**

- **功能描述**：商机规则配置（价值分层规则、生命周期规则等）
- **客户场景**：运营人员配置商机识别规则，系统自动识别商机

### 2.2 系统管理模块

#### 2.2.1 权限管理 (Auth Management)

**2.2.1.1 账号管理 (Account Management)**

- **功能描述**：系统账号的创建、编辑、删除、权限分配
- **客户场景**：系统管理员管理客户账号

**2.2.1.2 角色管理 (Role Management)**

- **功能描述**：角色的创建、编辑、删除，权限配置
- **核心功能**：
  - 基础信息（角色名称、角色级别、状态）
  - 功能权限（菜单权限、按钮权限）
  - 数据范围（全部数据、自定义部门、仅本人数据等）
  - 字段脱敏（配置敏感字段的脱敏规则）
- **客户场景**：
  - 系统管理员创建"运营专员"角色，配置功能权限和数据范围
  - 配置"客服"角色，设置字段脱敏规则（客服只能看到脱敏后的手机号）

**2.2.1.3 菜单管理 (Menu Management)**

- **功能描述**：系统菜单的配置和管理
- **客户场景**：系统管理员配置系统菜单结构

#### 2.2.2 系统管理 (System Management)

**2.2.2.1 部门管理 (Department Management)**

- **功能描述**：组织架构管理
- **客户场景**：系统管理员维护部门树结构

**2.2.2.2 通知公告 (Notice Management)**

- **功能描述**：系统通知公告的发布和管理
- **客户场景**：系统管理员发布系统公告

#### 2.2.3 系统监控 (System Monitor)

**2.2.3.1 操作日志 (Operation Log)**

- **功能描述**：记录客户操作日志，支持查询和审计
- **客户场景**：系统管理员查看客户操作记录，进行审计

**2.2.3.2 登录日志 (Login Log)**

- **功能描述**：记录客户登录日志
- **客户场景**：系统管理员查看客户登录记录，排查安全问题

## 3. 核心业务流程

### 3.1 客户数据管理流程

1. **数据接入** → 系统自动进行数据质量检测
2. **异常处理** → 异常中心处理数据质量问题
3. **身份统一** → OneID 统一客户身份（自动合并/人工审批）
4. **数据维护** → 客户列表进行客户维护和纠错反馈

### 3.2 客户分群与营销流程

1. **分群构建** → 客户筛选与分群模块构建分群规则
2. **分群审批** → 提交审批，等待审批通过
3. **分群管理** → 分群管理模块查看分群状态和成员
4. **商机转化** → 推送分群为商机，进入商机管理流程

### 3.3 商机管理流程

1. **规则配置** → 配置商机识别规则（价值分层、生命周期）
2. **商机识别** → 系统自动识别商机
3. **商机看板** → 查看商机整体概况
4. **商机跟进** → 商机列表进行跟进和管理

### 3.4 数据质量保障流程

1. **异常检测** → 系统自动检测数据质量问题
2. **异常分类** → 按异常类型分类（有效性、唯一性、完整性、一致性、合规性）
3. **异常处理** → 异常中心处理异常（快速编辑、合并/拆分、重试同步）
4. **审批流程** → 高风险操作需审批

## 4. 关键特性

### 4.1 数据安全与隐私

- **数据脱敏**：手机号、证件号等敏感信息自动脱敏显示
- **权限控制**：基于角色的功能权限、数据范围、字段脱敏配置
- **审计日志**：完整的操作日志和登录日志记录

### 4.2 数据质量保障

- **异常检测**：自动检测数据质量问题（有效性、唯一性、完整性、一致性、合规性）
- **冲突解决**：身份冲突智能合并/拆分，支持人工决策
- **纠错反馈**：客户可提交数据错误反馈

### 4.3 智能化能力

- **自动化标签**：基于规则的自动化标签生成
- **智能分群**：可视化规则构建，支持复杂逻辑组合
- **商机识别**：基于价值分层和生命周期的自动商机识别

### 4.4 客户体验

- **360度全景视图**：一站式查看客户全貌
- **可视化规则构建**：拖拽式规则编辑器
- **实时数据更新**：支持数据刷新和实时监控
- **多语言支持**：简体中文、繁体中文、英文

## 5. 性能要求

### 5.1 响应时间

- **页面响应**：< 20 秒
- **数据处理**：T+1（次日更新）

### 5.2 数据量

- **支持大规模数据**：百万级客户数据
- **分页查询**：支持大数据量分页

## 6. 客户角色与权限

### 6.1 角色定义

- **系统管理员**：系统配置、权限管理、日志查看
- **运营人员**：客户管理、分群管理、商机管理
- **数据分析师**：分群构建、标签管理、数据分析
- **客服人员**：客户查询、客户维护（受限权限）

### 6.2 权限控制

- **功能权限**：菜单权限、按钮权限
- **数据权限**：全部数据、部门数据、仅本人数据
- **字段权限**：字段脱敏配置
