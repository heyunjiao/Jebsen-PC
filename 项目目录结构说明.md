# Jebsen 项目目录结构说明文档

## 📋 项目概述

Jebsen 是一款基于 Vue3.4、TypeScript、Vite5、Pinia、Element-Plus 开发的后台管理系统框架。本文档详细说明了项目目录结构、各目录/文件的作用以及使用场景。

---

## 📁 根目录文件说明

### 配置文件

| 文件                 | 作用                   | 使用场景                                                      |
| -------------------- | ---------------------- | ------------------------------------------------------------- |
| `package.json`       | 项目依赖管理和脚本配置 | 管理 npm/pnpm 依赖包，定义项目脚本命令（dev、build、lint 等） |
| `pnpm-lock.yaml`     | 锁定依赖版本           | 确保团队成员安装相同版本的依赖包                              |
| `tsconfig.json`      | TypeScript 编译配置    | 配置 TS 编译选项、路径别名、类型检查规则                      |
| `vite.config.ts`     | Vite 构建工具配置      | 配置开发服务器、构建选项、插件、代理等                        |
| `postcss.config.cjs` | PostCSS 配置           | 配置 CSS 后处理（如 autoprefixer）                            |
| `index.html`         | 项目入口 HTML          | 应用的 HTML 模板，Vite 会注入构建产物                         |

### 代码规范配置

| 文件               | 作用                    | 使用场景                                 |
| ------------------ | ----------------------- | ---------------------------------------- |
| `.eslintrc.cjs`    | ESLint 代码检查配置     | 定义 JavaScript/TypeScript/Vue 代码规范  |
| `.eslintignore`    | ESLint 忽略文件         | 指定不需要进行代码检查的文件/目录        |
| `.prettierrc.cjs`  | Prettier 代码格式化配置 | 统一代码格式化规则（缩进、引号、分号等） |
| `.prettierignore`  | Prettier 忽略文件       | 指定不需要格式化的文件/目录              |
| `.stylelintrc.cjs` | Stylelint 样式检查配置  | 定义 CSS/SCSS 样式代码规范               |
| `.stylelintignore` | Stylelint 忽略文件      | 指定不需要进行样式检查的文件/目录        |
| `.editorconfig`    | 编辑器配置              | 统一不同编辑器的编码风格（缩进、换行等） |

### Git 相关配置

| 文件                     | 作用                 | 使用场景                                              |
| ------------------------ | -------------------- | ----------------------------------------------------- |
| `.gitignore`             | Git 忽略文件         | 指定不需要提交到 Git 的文件（node_modules、dist 等）  |
| `commitlint.config.cjs`  | Git 提交信息规范配置 | 规范 Git commit 信息格式（使用 conventional commits） |
| `lint-staged.config.cjs` | lint-staged 配置     | 在 Git 提交前对暂存文件执行 lint 检查                 |

### 其他文件

| 文件           | 作用         | 使用场景                       |
| -------------- | ------------ | ------------------------------ |
| `README.md`    | 项目说明文档 | 项目介绍、安装使用、功能特性等 |
| `CHANGELOG.md` | 项目更新日志 | 记录版本更新内容和变更历史     |

## 📁 根目录文件夹说明

### `.husky/`

**作用：** Git Hooks 配置目录  
**使用场景：**

- 配置 Git 提交前的钩子（pre-commit、commit-msg 等）
- 在提交代码前自动执行代码检查和格式化
- 确保代码质量符合团队规范

### `.vscode/`

**作用：** VSCode 编辑器推荐配置  
**使用场景：**

- 统一团队开发环境配置
- 推荐扩展、工作区设置等
- 提升开发体验和代码一致性

### `build/`

**作用：** Vite 构建相关配置文件  
**使用场景：**

- `getEnv.ts` - 环境变量处理工具
- `plugins.ts` - Vite 插件配置（ESLint、SVG、压缩等）
- `proxy.ts` - 开发环境代理配置
- 当需要修改构建配置、添加插件或配置代理时使用

### `public/`

**作用：** 静态资源目录（不会被 Vite 打包处理）  
**使用场景：**

- 存放不需要经过构建处理的静态文件
- 如：`logo.png`、`vue.svg` 等
- 这些文件会直接复制到构建输出目录

### `node_modules/`

**作用：** 项目依赖包目录  
**使用场景：**

- 存放通过 `pnpm install` 安装的所有依赖包
- 通常不需要手动操作，由包管理器自动管理

## 📁 src/ 目录结构详解

### `src/main.ts`

**作用：** 项目入口文件  
**使用场景：**

- 初始化 Vue 应用
- 注册全局插件（Element Plus、Router、Pinia、i18n 等）
- 注册全局指令和组件
- 挂载应用到 DOM

### `src/App.vue`

**作用：** 根组件  
**使用场景：**

- 应用的最外层组件
- 配置 Element Plus 的全局配置（语言、尺寸、主题等）
- 包含路由视图 `<router-view>`

## 📁 src/api/ - API 接口管理

**作用：** 统一管理所有 API 接口请求  
**使用场景：** 所有与后端交互的接口调用都在此目录

### `api/index.ts`

- Axios 实例创建和配置
- 请求/响应拦截器
- 统一错误处理

### `api/interface/index.ts`

- API 请求/响应的 TypeScript 类型定义
- 接口参数和返回值的类型约束

### `api/config/servicePort.ts`

- API 服务端口配置
- 不同环境的接口地址配置

### `api/helper/`

- `axiosCancel.ts` - 请求取消功能（防止重复请求）
- `checkStatus.ts` - HTTP 状态码检查和错误处理

### `api/modules/`

- `login.ts` - 登录相关接口
- `user.ts` - 客户相关接口
- `upload.ts` - 文件上传接口
- 按业务模块划分的接口文件

## 🎯 常用开发场景指南

### 1. 添加新页面

1. 在 `src/views/` 下创建页面组件
2. 在 `src/routers/modules/staticRouter.ts` 或 `dynamicRouter.ts` 中添加路由配置
3. 如需权限控制，在 `src/api/modules/` 中添加接口

### 2. 添加新 API 接口

1. 在 `src/api/modules/` 下创建或修改接口文件
2. 在 `src/api/interface/index.ts` 中定义类型
3. 在组件中使用 `import` 导入接口函数

### 3. 添加全局组件

1. 在 `src/components/` 下创建组件
2. 如需全局注册，在 `src/main.ts` 中注册

### 4. 添加自定义指令

1. 在 `src/directives/modules/` 下创建指令文件
2. 在 `src/directives/index.ts` 中注册

### 5. 添加状态管理

1. 在 `src/stores/modules/` 下创建 Store 文件
2. 在组件中使用 `useXxxStore()` 获取状态

### 6. 添加工具函数

1. 在 `src/utils/` 下创建工具文件
2. 在 `src/utils/index.ts` 中导出

### 7. 修改主题样式

1. 修改 `src/styles/var.scss` 中的变量
2. 修改 `src/styles/theme/` 下的主题配置文件

### 8. 配置代理

1. 修改 `build/proxy.ts` 中的代理配置
2. 在 `.env.development` 中配置 `VITE_PROXY`

## 📝 总结

本文档详细说明了 Jebsen 项目的目录结构和各文件/目录的作用。在开发过程中：

- **新增功能**：主要在 `src/views/`、`src/api/`、`src/components/` 下操作
- **修改配置**：主要在 `src/config/`、`vite.config.ts`、`.env.*` 下操作
- **全局功能**：主要在 `src/stores/`、`src/routers/`、`src/utils/` 下操作
- **样式主题**：主要在 `src/styles/` 下操作
