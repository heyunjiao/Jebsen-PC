## 菜单配置

### 方式一：通过菜单管理界面添加

1. 登录系统后，进入 **系统管理** -> **菜单管理**

2. 添加父菜单（系统监控）：

   - **菜单名称**: 系统监控
   - **菜单类型**: 目录 (M)
   - **显示排序**: 根据实际情况设置（建议放在系统管理下）
   - **路由地址**: `monitor`
   - **菜单图标**: 根据实际情况选择（如：`Monitor` 或 `View`）
   - **状态**: 正常

3. 添加子菜单 - 参数设置：

   - **上级菜单**: 选择"系统管理"（或根据实际情况选择）
   - **菜单名称**: 参数设置
   - **菜单类型**: 菜单 (C)
   - **显示排序**: 1
   - **路由地址**: `/system/configManage/index`
   - **组件路径**: `system/configManage/index`
   - **路由名称**: `configManage`
   - **菜单图标**: `Setting` 或 `Tools`
   - **权限字符**: `system:config:list`
   - **状态**: 正常

4. 添加子菜单 - 通知公告：

   - **上级菜单**: 选择"系统管理"
   - **菜单名称**: 通知公告
   - **菜单类型**: 菜单 (C)
   - **显示排序**: 2
   - **路由地址**: `/system/noticeManage/index`
   - **组件路径**: `system/noticeManage/index`
   - **路由名称**: `noticeManage`
   - **菜单图标**: `Bell` 或 `Message`
   - **权限字符**: `system:notice:list`
   - **状态**: 正常

5. 添加子菜单 - 操作日志：

   - **上级菜单**: 选择"系统监控"（或"系统管理"）
   - **菜单名称**: 操作日志
   - **菜单类型**: 菜单 (C)
   - **显示排序**: 1
   - **路由地址**: `/system/operlogManage/index`
   - **组件路径**: `system/operlogManage/index`
   - **路由名称**: `operlogManage`
   - **菜单图标**: `Document` 或 `List`
   - **权限字符**: `monitor:operlog:list`
   - **状态**: 正常

6. 添加子菜单 - 登录日志：
   - **上级菜单**: 选择"系统监控"（或"系统管理"）
   - **菜单名称**: 登录日志
   - **菜单类型**: 菜单 (C)
   - **显示排序**: 2
   - **路由地址**: `/system/loginlogManage/index`
   - **组件路径**: `system/loginlogManage/index`
   - **路由名称**: `loginlogManage`
   - **菜单图标**: `User` 或 `Calendar`
   - **权限字符**: `monitor:logininfor:list`
   - **状态**: 正常

## 功能说明

### 参数设置模块

- **功能**: 管理系统参数配置
- **主要字段**:
  - 参数名称
  - 参数键名
  - 参数键值
  - 系统内置（是/否）
  - 备注
- **操作**: 新增、修改、删除、导出、刷新缓存

### 通知公告模块

- **功能**: 管理系统通知和公告
- **主要字段**:
  - 公告标题
  - 公告类型（通知/公告）
  - 状态（正常/关闭）
  - 公告内容（富文本编辑器）
  - 创建者
- **操作**: 新增、修改、删除

### 操作日志模块

- **功能**: 查看系统操作日志
- **主要字段**:
  - 系统模块
  - 操作类型（新增/修改/删除等）
  - 操作人员
  - 操作地址
  - 操作地点
  - 操作状态（正常/失败）
  - 操作时间
  - 消耗时间
- **操作**: 查看详细、删除、清空、导出

### 登录日志模块

- **功能**: 查看客户登录日志
- **主要字段**:
  - 客户名称
  - 登录地址
  - 登录地点
  - 浏览器
  - 操作系统
  - 登录状态（成功/失败）
  - 操作信息
  - 登录日期
- **操作**: 删除、清空、解锁客户、导出

## 技术实现

### 使用的技术栈

- **Vue 3** + **TypeScript**
- **Element Plus** UI 组件库
- **ProTable** 表格组件
- **Composition API**

### 特点

1. ✅ 使用 ProTable 组件，支持搜索、分页、排序等功能
2. ✅ 响应式设计，适配不同屏幕尺寸
3. ✅ 表单验证，确保数据完整性
4. ✅ 支持批量操作（批量删除）
5. ✅ 使用 TypeScript，类型安全
6. ✅ 统一的 API 调用方式

## API 接口说明

所有 API 接口已创建在对应的文件中：

- `src/api/system/config.ts` - 参数配置 API
- `src/api/system/notice.ts` - 通知公告 API
- `src/api/monitor/operlog.ts` - 操作日志 API
- `src/api/monitor/logininfor.ts` - 登录日志 API

## 注意事项

1. 菜单配置后需要刷新页面或重新登录才能看到新菜单
2. 确保路由路径和组件路径正确
3. 权限字符需要与后端权限系统对应
4. 图标名称需要是 Element Plus 支持的图标名称
5. 操作日志和登录日志模块建议放在"系统监控"父菜单下
6. 参数设置和通知公告模块可以放在"系统管理"父菜单下

## 后续调整

根据实际需求，可以在菜单管理界面中调整：

- 菜单位置（通过显示排序调整）
- 菜单图标
- 菜单名称
- 权限配置
- 父菜单归属（可以调整到不同的父菜单下）

## 路由说明

由于项目使用动态路由（从菜单数据生成），不需要在静态路由中添加。只需在菜单管理中配置菜单项，系统会自动生成路由。
