## 三、数据权限配置表格

### 3.1 数据范围权限配置

| 角色名称   | 数据范围类型         | 数据范围说明                       | 适用场景                       |
| ---------- | -------------------- | ---------------------------------- | ------------------------------ |
| 系统管理员 | 全部数据权限         | 可访问系统内所有数据，无任何限制   | 系统配置、全局数据管理         |
| 运营经理   | 本部门及以下数据权限 | 可访问所在部门及下级部门的所有数据 | 运营部门经理管理团队数据       |

| 数据分析师 | 全部数据权限         | 可访问系统内所有数据，用于数据分析 | 数据分析、报表生成             |
| 营销经理   | 本部门及以下数据权限 | 可访问所在部门及下级部门的所有数据 | 营销部门经理管理团队数据       |
| 营销专员   | 本部门数据权限       | 仅可访问所在部门的数据             | 营销部门普通员工处理部门数据   |
| 销售经理   | 本部门及以下数据权限 | 可访问所在部门及下级部门的所有数据 | 销售部门经理管理团队数据       |
| 销售人员   | 仅本人数据权限       | 仅可访问自己创建或分配的数据       | 销售人员查看自己的客户和商机   |
| 客服人员   | 仅本人数据权限       | 仅可访问自己处理的客户数据         | 客服人员查看自己处理的客户信息 |
| 审计人员   | 全部数据权限         | 可访问系统内所有数据，用于审计追溯 | 操作日志、登录日志审计         |

### 3.2 数据范围详细说明

| 数据范围类型         | 数据范围标识     | 数据访问规则                       | 示例说明                           |
| -------------------- | ---------------- | ---------------------------------- | ---------------------------------- |
| 全部数据权限         | `all`            | 可访问系统内所有数据，无任何限制   | 系统管理员可查看所有客户、所有商机 |
| 自定义数据权限       | `custom`         | 可访问指定部门或指定范围的数据     | 可配置访问特定经销商的数据         |
| 本部门及以下数据权限 | `dept_and_below` | 可访问所在部门及所有下级部门的数据 | 大区经理可查看大区及所有门店数据   |
| 本部门数据权限       | `dept_only`      | 仅可访问所在部门的数据             | 店长可查看本店数据                 |
| 仅本人数据权限       | `self_only`      | 仅可访问自己创建或分配的数据       | 销售人员只能查看自己的客户和商机   |

## 五、权限配置推荐方案

### 5.1 功能权限推荐配置

#### 5.1.1 核心业务角色权限推荐

**运营人员（运营经理、运营专员）**

- ✅ 必须权限：顾客列表、标签管理、分群管理、异常中心、审批中心
- ✅ 推荐权限：数据监控看板、顾客筛选与分群、源数据采集与质量门禁工作台
- ❌ 不推荐权限：商机管理、系统管理、权限管理

**营销人员（营销经理、营销专员）**

- ✅ 必须权限：顾客列表、顾客筛选与分群、分群管理、商机管理
- ✅ 推荐权限：数据监控看板、商机看板、商机列表
- ❌ 不推荐权限：异常中心、标签管理、系统管理

**销售人员（销售经理、销售人员）**

- ✅ 必须权限：顾客列表、商机管理（看板、列表、追踪）
- ✅ 推荐权限：分群管理（仅查看）
- ❌ 不推荐权限：异常中心、审批中心、标签管理、系统管理

#### 5.1.2 管理角色权限推荐

**系统管理员**

- ✅ 全部权限：系统所有功能模块和操作权限

**部门经理（运营经理、营销经理、销售经理）**

- ✅ 管理权限：本部门数据管理、团队数据查看、审批权限
- ✅ 业务权限：对应业务模块的完整权限

### 5.2 数据权限推荐配置

#### 5.2.1 数据范围推荐

| 组织层级  | 推荐数据范围         | 说明                   |
| --------- | -------------------- | ---------------------- |
| 集团/总部 | 全部数据权限         | 系统管理员、数据分析师 |
| 大区/区域 | 本部门及以下数据权限 | 大区经理、区域经理     |
| 门店/部门 | 本部门数据权限       | 店长、部门经理         |
| 个人      | 仅本人数据权限       | 销售人员、客服人员     |

#### 5.2.2 数据权限配置原则

1. **最小权限原则**：默认授予最小必要权限，根据实际需求逐步开放
2. **职责分离原则**：不同角色职责不同，数据访问范围应严格区分
3. **层级管理原则**：上级可查看下级数据，下级不可查看上级数据
4. **业务需求原则**：根据业务实际需求配置数据权限，避免过度授权

### 5.3 字段脱敏推荐配置

#### 5.3.1 敏感字段脱敏推荐

| 字段类型   | 敏感级别 | 推荐脱敏角色                             | 脱敏规则         |
| ---------- | -------- | ---------------------------------------- | ---------------- |
| 手机号     | 高敏感   | 运营专员、数据分析师、营销专员、客服人员 | 掩码（中间4位）  |
| 身份证号   | 极高敏感 | 除系统管理员外所有角色                   | 掩码（中间部分） |
| VIN码      | 中敏感   | 客服人员、审计人员                       | 掩码（部分）     |
| 车牌号     | 中敏感   | 客服人员、审计人员                       | 掩码（部分）     |
| 年消费金额 | 低敏感   | 客服人员                                 | 掩码（部分）     |

#### 5.3.2 脱敏配置原则

1. **业务必需原则**：销售人员需要完整手机号进行业务联系，可明文显示
2. **最小暴露原则**：非业务必需角色，敏感信息应进行脱敏处理
3. **合规要求原则**：符合数据保护法规要求，保护客户隐私
4. **可追溯原则**：审计人员需要查看完整信息，但应进行脱敏处理

## 六、权限配置实施建议

### 6.1 权限配置流程

1. **角色定义**：根据组织架构和业务需求，定义系统角色
2. **权限梳理**：梳理各角色的功能权限、数据权限、字段权限需求
3. **权限配置**：在角色管理模块中配置各角色的权限
4. **权限测试**：创建测试账号，验证权限配置是否正确
5. **权限发布**：权限配置确认后，正式发布使用
6. **权限审计**：定期审计权限配置，确保权限合理有效

### 6.2 权限配置注意事项

1. **权限最小化**：遵循最小权限原则，避免过度授权
2. **权限分离**：不同角色权限应明确区分，避免权限重叠
3. **定期审查**：定期审查权限配置，及时调整不合理权限
4. **权限变更**：权限变更需经过审批流程，记录变更日志
5. **权限培训**：对客户进行权限使用培训，确保正确使用权限

### 8.2 权限配置常见问题

**Q1: 如何为新员工配置权限？**  
A: 根据员工岗位和职责，在账号管理中创建账号，并分配对应的角色。系统会自动应用该角色的所有权限配置。

**Q2: 如何修改角色的权限？**  
A: 在角色管理模块中，选择对应角色，进入权限配置页面，修改功能权限、数据权限或字段脱敏配置，保存后立即生效。

**Q3: 为什么我看不到某些菜单？**  
A: 请检查您的角色权限配置，确认是否已分配该菜单的访问权限。如需要，请联系系统管理员申请权限。

**Q4: 为什么我看到的手机号是脱敏的？**  
A: 根据您的角色权限配置，手机号字段被设置为脱敏显示。这是为了数据安全考虑。如业务需要完整手机号，请联系系统管理员调整权限配置。

**Q5: 如何申请新的权限？**  
A: 请联系系统管理员，说明需要申请的权限和业务需求。管理员会根据实际情况进行权限分配。
