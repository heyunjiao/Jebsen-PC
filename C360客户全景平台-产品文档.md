## 1. 项目概述

### 1.1 项目背景

C360 客户全景平台是一个面向 B 端企业的客户数据管理平台，旨在帮助企业实现客户数据的统一管理、智能分群、精准营销和商机转化。

### 1.2 产品定位

- **核心价值**：统一客户身份（OneID）、数据质量保障、智能客户分群、商机管理
- **目标客户**：企业运营人员、数据分析师、营销人员、系统管理员
- **应用场景**：客户数据管理、精准营销、商机挖掘、数据质量治理

### 1.3 技术架构

- **前端框架**：Vue 3.4+ (Composition API)
- **UI 组件库**：Element Plus 2.7+
- **状态管理**：Pinia
- **构建工具**：Vite 5+
- **国际化**：支持简体中文、繁体中文、英文

#### 2.2.2 典型业务场景流程

##### 场景一：新客户数据接入与处理

**业务背景**：新客户数据从外部系统接入，需要经过质量检测、身份统一、客户建立等流程。

```
1. 数据接入
   └─ 源数据采集与质量门禁工作台
      ├─ 配置数据源连接
      ├─ 设置采集规则
      └─ 启动采集任务

2. 质量检测（自动）
   └─ 系统自动检测
      ├─ 发现手机号格式错误 → 异常中心（有效性异常）
      ├─ 发现重复数据 → 异常中心（唯一性异常）
      └─ 发现必填字段缺失 → 异常中心（完整性异常）

3. 异常处理
   └─ 异常中心
      ├─ 数据质量专员查看异常列表
      ├─ 快速编辑修复格式错误
      ├─ 处理重复数据（合并或标记）
      └─ 补充缺失字段

4. 身份统一（自动/人工）
   └─ 系统自动识别
      ├─ 发现相同身份 → 自动合并（低风险）或提交审批（高风险）
      └─ 生成/更新 OneID

5. 客户建立
   └─ 客户列表
      ├─ 系统自动创建客户客户
      ├─ 关联标签（基于规则自动打标签）
      └─ 建立360度全景视图

6. 数据监控
   └─ Dashboard 数据监控看板
      └─ 实时更新客户统计数据
```

##### 场景二：精准营销分群创建

**业务背景**：营销人员需要创建"高价值潜在客户"分群，用于精准营销活动。

```
1. 分群规则构建
   └─ 客户筛选与分群
      ├─ 选择"漏斗筛选"模式
      ├─ 设置筛选条件：
      │  ├─ 基础属性：年龄 25-45岁
      │  ├─ 客户价值：消费金额 > 10万
      │  ├─ 客户行为：最近30天有浏览行为
      │  └─ 标签：包含"潜在客户"标签
      └─ 系统进行冲突检测

2. 分群预估
   └─ 系统计算
      ├─ 预估人数：1,234人
      ├─ 覆盖率：5.6%
      └─ 显示预估结果

3. 样本预览
   └─ 查看样本数据
      ├─ 预览前10条客户数据
      └─ 验证规则准确性

4. 洞察分析
   └─ 洞察面板
      ├─ 分群特征分析
      ├─ 特征分布统计
      └─ 可视化图表展示

5. 保存为分群
   └─ 提交审批
      ├─ 填写分群名称："高价值潜在客户"
      ├─ 填写分群描述
      └─ 提交审批请求

6. 审批流程
   └─ 审批中心
      ├─ 审批人收到审批通知
      ├─ 查看分群规则和预估结果
      ├─ 评估风险（置信度评估）
      └─ 批准/驳回

7. 分群生效
   └─ 分群管理
      ├─ 分群状态：计算中 → 生效
      ├─ 查看分群成员列表
      └─ 查看成员数统计

8. 推送为商机（可选）
   └─ 分群管理
      ├─ 选择分群
      ├─ 点击"推送为商机"
      └─ 系统自动创建商机

9. 商机管理
   └─ 商机列表
      ├─ 查看新创建的商机
      ├─ 分配给销售人员
      └─ 开始跟进
```

##### 场景三：商机识别与跟进

**业务背景**：系统根据配置的规则自动识别商机，销售人员跟进转化。

```
1. 规则配置（系统管理员）
   └─ 规则配置
      ├─ 配置价值分层规则：
      │  ├─ 钻石：消费金额 > 50万
      │  ├─ 黄金：消费金额 20-50万
      │  └─ 白银：消费金额 10-20万
      │
      └─ 配置生命周期规则：
         ├─ 潜在客户：首次接触
         ├─ 意向客户：有明确需求
         ├─ 成交客户：已完成交易
         └─ 流失客户：长期无互动

2. 商机自动识别（系统自动）
   └─ 系统定时任务
      ├─ 扫描客户数据
      ├─ 匹配价值分层规则
      ├─ 匹配生命周期规则
      └─ 自动创建商机

3. 商机看板监控
   └─ 商机看板
      ├─ 查看价值分层统计
      │  ├─ 钻石：50个
      │  ├─ 黄金：200个
      │  └─ 白银：500个
      │
      └─ 查看生命周期统计
         ├─ 潜在客户：300个
         ├─ 意向客户：250个
         └─ 成交客户：200个

4. 商机分配
   └─ 商机列表
      ├─ 筛选高价值商机（钻石/黄金）
      ├─ 分配给对应销售人员
      └─ 设置跟进提醒

5. 商机跟进
   └─ 商机列表
      ├─ 销售人员查看分配的商机
      ├─ 查看客户360度全景视图
      ├─ 添加跟进记录
      ├─ 更新商机状态
      └─ 记录跟进结果

6. 商机转化分析
   └─ 商机追踪
      ├─ 查看转化率统计
      ├─ 分析转化漏斗
      ├─ 评估跟进效果
      └─ 优化跟进策略
```

##### 场景四：数据质量保障流程

**业务背景**：系统检测到客户身份冲突，需要人工决策合并或拆分。

```
1. 异常检测（系统自动）
   └─ 系统检测
      ├─ 发现同一手机号对应多个客户客户
      ├─ 发现同一身份证号对应多个客户客户
      └─ 标记为"身份冲突"异常

2. 异常分类
   └─ 异常中心
      ├─ 异常类型：唯一性异常
      ├─ 异常级别：高风险（需审批）
      └─ 显示在"待处理"列表

3. 冲突详情查看
   └─ 异常中心
      ├─ 点击异常任务
      ├─ 查看冲突详情：
      │  ├─ 客户A：姓名"张三"，手机"138****5678"
      │  ├─ 客户B：姓名"张三三"，手机"138****5678"
      │  └─ 系统置信度：85%（建议合并）
      └─ 查看关联数据（订单、标签、分群等）

4. 身份冲突解决
   └─ 身份冲突解决工作台
      ├─ 数据质量专员打开工作台
      ├─ 查看系统建议（合并/拆分）
      ├─ 人工决策：
      │  ├─ 确认是同一人 → 选择合并
      │  └─ 确认是不同人 → 选择拆分
      └─ 填写处理意见

5. 提交审批（高风险操作）
   └─ 审批中心
      ├─ 自动创建审批任务
      ├─ 审批人收到通知
      ├─ 查看合并/拆分详情
      ├─ 评估风险（置信度、影响范围）
      └─ 批准/驳回

6. 执行操作
   └─ 系统自动执行
      ├─ 合并操作：
      │  ├─ 合并客户客户
      │  ├─ 统一 OneID
      │  ├─ 合并订单数据
      │  ├─ 合并标签
      │  └─ 合并分群
      │
      └─ 拆分操作：
         ├─ 拆分客户客户
         ├─ 生成新的 OneID
         └─ 分配数据归属

7. 结果反馈
   └─ 异常中心
      ├─ 异常状态：已处理
      ├─ 处理结果：已合并/已拆分
      └─ 更新客户客户
```

### 2.4 系统协作机制

#### 2.4.1 自动化流程

**1. 数据质量自动化检测**
- **触发时机**：数据接入时、定时任务
- **执行模块**：源数据采集与质量门禁工作台 → 异常中心
- **处理方式**：自动检测 → 自动分类 → 自动标记异常

**2. 标签自动化打标**
- **触发时机**：客户数据更新时、定时任务
- **执行模块**：标签管理 → 客户列表
- **处理方式**：规则匹配 → 自动打标签 → 更新客户客户

**3. 商机自动识别**
- **触发时机**：定时任务（T+1）
- **执行模块**：规则配置 → 商机列表
- **处理方式**：规则匹配 → 自动创建商机 → 更新商机看板

**4. 分群自动计算**
- **触发时机**：分群规则保存后、定时任务
- **执行模块**：分群管理
- **处理方式**：规则计算 → 生成分群成员 → 更新统计

#### 2.4.2 人工干预流程

**1. 异常处理流程**
- **触发时机**：系统检测到异常
- **执行模块**：异常中心 → 审批中心（高风险）
- **处理方式**：人工查看 → 人工处理 → 审批确认（如需）

**2. 分群创建流程**
- **触发时机**：客户创建分群
- **执行模块**：客户筛选与分群 → 审批中心 → 分群管理
- **处理方式**：人工构建规则 → 提交审批 → 审批通过 → 分群生效

**3. 商机跟进流程**
- **触发时机**：商机创建后
- **执行模块**：商机列表 → 商机追踪
- **处理方式**：人工分配 → 人工跟进 → 记录结果 → 分析转化

### B. 常见问题

#### B.1 数据脱敏相关问题

**Q: 如何配置字段脱敏规则？**  
A: 在角色管理中，选择对应角色，进入"字段脱敏配置"页面，配置需要脱敏的字段和脱敏级别。

**Q: 为什么我看不到完整的手机号？**  
A: 根据您的角色权限，手机号字段被配置为脱敏显示，这是为了数据安全考虑。

#### B.2 权限相关问题

**Q: 为什么我无法访问某个菜单？**  
A: 请联系系统管理员，检查您的角色权限配置，确认是否已分配该菜单的访问权限。

**Q: 如何申请新的权限？**  
A: 请联系系统管理员，说明需要申请的权限和业务需求，管理员会根据实际情况进行权限分配。

#### B.3 数据查询相关问题

**Q: 为什么查询结果为空？**  
A: 请检查：
1. 筛选条件是否正确
2. 数据范围权限是否包含目标数据
3. 数据是否已被删除或停用

**Q: 如何导出大量数据？**  
A: 系统支持分页导出，建议分批导出，避免单次导出数据量过大导致系统响应缓慢。
