## 1. 产品概述

### 1.1 模块定位

审批中心模块是 C360 客户全景平台的高风险操作审批流程管理模块，负责数据合并、分群创建等高风险操作的审批流程。支持待审批列表、快速审批、审批详情查看、风险提示和置信度评估等功能。

### 1.2 业务价值

- **风险控制**：对高风险操作进行二次确认，降低数据风险
- **流程规范**：规范审批流程，确保操作可追溯
- **智能评估**：基于置信度自动判断是否需要审批
- **高效审批**：提供快速审批功能，提高审批效率

### 1.3 目标客户

- **审批人**：系统管理员、数据质量负责人
- **申请人**：运营人员、数据分析师

## 4. 业务规则

### 4.1 审批触发规则

#### 4.1.1 自动触发规则
- **置信度 < 95%**：自动提交审批
- **包含人工修改**：自动提交审批
- **关键字段变更**：自动提交审批
- **高风险操作**：数据合并、分群创建等

#### 4.1.2 手动触发规则
- **客户主动提交**：客户可主动提交审批请求
- **强制审批**：某些操作强制需要审批

### 4.2 置信度计算规则

#### 4.2.1 计算因素
- **匹配字段数量**：匹配的字段越多，置信度越高
- **数据来源可靠性**：数据来源越可靠，置信度越高
- **历史合并记录**：历史合并成功率越高，置信度越高
- **字段一致性**：字段一致性越高，置信度越高

#### 4.2.2 置信度阈值
- **默认阈值**：95%
- **可配置**：支持通过规则配置调整阈值
- **阈值应用**：低于阈值的操作需要审批

### 4.3 审批流程规则

#### 4.3.1 审批流程
```
提交审批 → 待审批 → 审批中 → 已批准/已驳回
```

#### 4.3.2 审批规则
- **单级审批**：一个审批人即可完成审批
- **审批时限**：支持设置审批时限（可选）
- **审批通知**：审批结果通知申请人

### 4.4 风险提示规则

#### 4.4.1 风险等级
- **高风险**：置信度 < 80%，关键字段变更
- **中风险**：置信度 80-95%，非关键字段变更
- **低风险**：置信度 >= 95%，无需审批

#### 4.4.2 风险提示
- **置信度显示**：显示置信度分数和等级
- **风险字段高亮**：风险字段高亮显示
- **建议操作**：根据置信度提供建议操作

## 5. 交互设计

### 5.1 页面布局

#### 5.1.1 Tab切换区域
- **位置**：页面顶部
- **布局**：Tab切换（待我审批/已处理）

- **位置**：Tab下方

  - 状态筛选
  - 快速审批按钮

### 5.2 审批详情对话框

#### 5.2.1 详情入口
- **触发方式**：点击审批列表中的"查看详情"按钮

- **左侧区域**：源数据信息

  - 变更字段高亮
  - 风险字段标记
- **右侧区域**：目标数据信息
  - 目标数据名称
  - 目标数据字段值
- **底部区域**：操作按钮
  - 批准按钮
  - 驳回按钮
  - 驳回原因输入

- **字段分组**：按字段分组展示
- **变更标识**：变更字段显示变更标识
- **风险标识**：风险字段显示风险标识
- **人工修改标识**：人工修改字段显示标识

### 5.3 快速审批

#### 5.3.1 快速批准
- **触发方式**：点击列表中的"批准"按钮
- **确认方式**：确认对话框
- **处理结果**：审批状态更新为"已批准"

#### 5.3.2 快速驳回
- **触发方式**：点击列表中的"驳回"按钮
- **确认方式**：输入驳回原因对话框
- **处理结果**：审批状态更新为"已驳回"

### 5.4 规则配置

#### 5.4.1 配置入口
- **触发方式**：点击"规则配置"按钮
- **打开方式**：弹窗（Dialog）
- **弹窗宽度**：800px

#### 5.4.2 配置表单
- **规则列表**：显示所有审批规则
- **阈值编辑**：支持编辑置信度阈值
- **启用/停用**：支持启用/停用规则
- **保存按钮**：保存配置

## 6. 异常处理

### 6.1 数据加载异常

#### 6.1.1 网络错误
- **场景**：网络请求失败
- **处理**：显示错误提示，提供重试按钮

#### 6.1.2 数据为空
- **场景**：查询结果为空
- **处理**：显示空状态提示

### 6.2 审批操作异常

#### 6.2.1 审批失败
- **场景**：审批操作失败
- **处理**：
  - 显示错误提示
  - 保留审批表单数据
  - 允许重新提交

#### 6.2.2 权限不足
- **场景**：无审批权限
- **处理**：
  - 显示权限提示
  - 隐藏审批按钮

## 10. 接口定义

### 10.1 获取审批列表

**接口路径**：`GET /api/approval/list`

**响应数据**：
```typescript
{
  code: number;
  msg: string;
  data: ApprovalRequest[];
}
```

### 10.2 获取审批详情

**接口路径**：`GET /api/approval/detail/:reqId`

**响应数据**：
```typescript
{
  code: number;
  msg: string;
  data: ApprovalDetailResponse;
}
```

### 10.3 批准请求

**接口路径**：`POST /api/approval/approve/:reqId`

### 10.4 驳回请求

**接口路径**：`POST /api/approval/reject/:reqId`

**请求参数**：
```typescript
{
  reason: string;
}
```

### 10.5 获取规则列表

**接口路径**：`GET /api/approval/rule/list`

**响应数据**：
```typescript
{
  code: number;
  msg: string;
  data: {
    list: RuleItem[];
    total: number;
  };
}
```

### 10.6 保存规则配置

**接口路径**：`POST /api/approval/rule/save/:ruleId`

**请求参数**：
```typescript
{
  threshold: number;
}
```
