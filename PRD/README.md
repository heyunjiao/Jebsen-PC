## PRD 文档标准

每个 PRD 文档应包含以下章节：

1. **产品概述**
   - 模块定位
   - 业务价值
   - 目标客户

2. **功能需求**
   - 功能清单
   - 功能优先级

3. **字段定义**
   - 所有字段的详细定义
   - 数据类型、是否必填、说明、示例

4. **业务规则**
   - 详细的业务规则说明
   - 触发条件、处理方式、异常处理

5. **交互设计**
   - 页面布局
   - 交互流程
   - 交互细节

6. **异常处理**
   - 各种异常场景的处理方式

7. **权限控制**
   - 功能权限
   - 数据范围权限

8. **数据统计**
   - 统计指标定义

9. **性能要求**
   - 响应时间要求
   - 数据量支持

10. **接口定义**
    - API 接口详细定义

11. **测试用例**
    - 功能测试用例
    - 异常测试用例

12. **更新日志**
    - 版本更新记录

## 创建新 PRD 文档步骤

1. **复制模板**：复制 `PRD模板.md` 文件
2. **重命名**：按照命名规范重命名，如 `04-分群管理模块-PRD.md`
3. **填写内容**：
   - 参考代码中的 interface 定义填写字段
   - 根据实际业务逻辑填写业务规则
   - 根据实际UI设计填写交互设计
   - 参考 API 文件填写接口定义
4. **检查完整性**：确保所有章节都已填写完整
5. **更新目录**：在本 README 中更新文档状态

## 命名规范

- 文件名格式：`[编号]-[模块名称]模块-PRD.md`
- 编号：两位数字，按模块顺序编号
- 模块名称：使用中文，与产品文档中的模块名称保持一致

## 维护说明

- **文档更新**：模块功能变更时，及时更新对应的 PRD 文档
- **版本管理**：每次重大更新时，更新文档版本号和更新日志
- **状态标记**：使用 ✅ 表示已完成，⏳ 表示待创建，🔄 表示更新中
