## 1. 产品概述

### 1.1 模块定位

客户列表模块是 C360 客户全景平台的核心模块，提供客户客户的统一管理、360度全景视图、客户维护、纠错反馈等功能。基于 OneID 统一身份标识，实现跨系统客户数据的整合与展示。

### 1.2 业务价值

- **统一客户视图**：基于 OneID 实现跨系统客户数据的统一展示
- **数据质量保障**：支持数据脱敏、冲突检测、纠错反馈
- **高效管理**：提供多条件筛选、批量操作、数据导出等功能
- **全景洞察**：360度全景视图展示客户全生命周期数据

### 1.3 目标客户

- **运营人员**：客户数据查询、客户维护、数据导出
- **数据分析师**：客户数据分析、数据质量检查
- **营销人员**：目标客户筛选、客户信息查看

## 4. 业务规则

### 4.1 数据脱敏规则

#### 4.1.1 手机号脱敏
- **规则**：显示前3位和后4位，中间4位用 `****` 替代
- **示例**：`13812345678` → `138****5678`
- **权限控制**：基于角色的字段脱敏配置

#### 4.1.2 证件号脱敏
- **规则**：显示前4位和后4位，中间用 `*` 替代
- **示例**：`320123199001011234` → `3201********1234`
- **权限控制**：基于角色的字段脱敏配置

#### 4.1.3 车牌号脱敏
- **规则**：显示前1位和后1位，中间用 `****` 替代
- **示例**：`沪A12345` → `沪A****5`
- **权限控制**：基于角色的字段脱敏配置

### 4.2 多值字段处理规则

#### 4.2.1 多值字段展示
- **规则**：支持一个字段多个值，每个值显示来源系统
- **展示方式**：
  - 列表模式：显示所有值，标注来源
  - 卡片模式：主值突出显示，其他值折叠
- **主值确定**：系统根据数据来源优先级确定主值

#### 4.2.2 多值字段编辑
- **规则**：支持添加、删除、修改多值字段
- **验证**：手机号格式验证、唯一性验证
- **保存**：保存时更新主值标记

### 4.3 冲突检测规则

#### 4.3.1 冲突识别
- **规则**：系统自动检测跨源数据冲突
- **冲突类型**：
  - 同一手机号对应多个客户客户
  - 同一证件号对应多个客户客户
  - 同一车牌号对应多个客户客户
- **冲突标记**：`hasConflict = true`，显示冲突警告标签

#### 4.3.2 冲突解决
- **规则**：通过异常中心处理冲突
- **解决方式**：
  - 自动合并（低风险）
  - 人工审批合并（高风险）
  - 拆分处理

### 4.4 生命周期状态规则

#### 4.4.1 状态定义
- **active（活跃）**：客户在90天内有交易或互动
- **inactive（非活跃）**：客户90天内无交易或互动
- **pending（待处理）**：客户数据待处理或待审核
- **conflict（冲突）**：客户数据存在冲突

#### 4.4.2 状态转换
- **自动转换**：系统根据业务规则自动更新状态
- **手动转换**：运营人员可手动调整状态
- **状态记录**：记录状态变更历史

## 6. 异常处理

### 6.1 数据加载异常

#### 6.1.1 网络错误
- **场景**：网络请求失败
- **处理**：
  - 显示错误提示："网络错误，请稍后重试"
  - 提供"重试"按钮
  - 记录错误日志

#### 6.1.2 数据为空
- **场景**：查询结果为空
- **处理**：
  - 显示空状态提示："暂无数据"
  - 提供"清除筛选条件"按钮
  - 显示筛选条件说明

### 6.2 数据操作异常

#### 6.2.1 保存失败
- **场景**：编辑保存失败
- **处理**：
  - 显示错误提示："保存失败：{错误信息}"
  - 保留表单数据，允许重新提交
  - 记录错误日志

#### 6.2.2 权限不足
- **场景**：无权限执行操作
- **处理**：
  - 显示权限提示："您没有权限执行此操作"
  - 隐藏或禁用相关操作按钮
  - 记录权限检查日志

### 6.3 数据冲突处理

#### 6.3.1 冲突检测
- **场景**：检测到数据冲突
- **处理**：
  - 显示冲突警告标签
  - 点击标签跳转到异常中心
  - 在360度视图中展示冲突详情

#### 6.3.2 冲突解决
- **场景**：冲突已解决
- **处理**：
  - 自动刷新列表
  - 移除冲突标签
  - 更新冲突状态

## 10. 接口定义

**接口路径**：`POST /api/customer/list`

**请求参数**：
```typescript
{
  pageNum: number;        // 页码
  pageSize: number;       // 每页数量
  oneId?: string;         // OneID
  name?: string;          // 姓名
  phone?: string;        // 手机号
  lifecycleStatus?: string; // 生命周期状态
  tags?: string[];       // 标签列表
  // ... 其他筛选条件
}
```

**响应数据**：
```typescript
{
  code: number;
  msg: string;
  data: {
    list: Customer[];
    total: number;
  };
}
```
