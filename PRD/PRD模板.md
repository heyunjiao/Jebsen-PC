1. **复制模板**：复制此模板文件，重命名为 `[模块编号]-[模块名称]模块-PRD.md`
2. **填写内容**：根据实际模块情况填写各个章节内容
3. **字段定义**：参考代码中的 interface 定义填写字段
4. **业务规则**：根据实际业务逻辑填写规则
5. **交互设计**：根据实际UI设计填写交互细节
6. **接口定义**：参考 API 文件填写接口定义

## 注意事项

- 所有字段定义必须与实际代码中的 interface 保持一致
- 业务规则必须详细说明，包括触发条件、处理方式、异常处理
- 交互设计必须包含页面布局、交互流程、异常处理
- 权限控制必须明确功能权限和数据范围权限
- 性能要求必须明确响应时间和数据量支持
