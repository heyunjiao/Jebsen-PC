## 1. 产品概述

### 1.1 模块定位

欢迎页是 C360 客户全景平台的首页入口，为客户提供个性化的欢迎信息、快捷导航入口、关键业务指标概览和系统状态监控。根据客户角色（管理员/业务客户）展示不同的内容，帮助客户快速了解系统状态和业务概况。

### 1.2 业务价值

- **提升客户体验**：个性化欢迎信息，根据时间段和客户角色定制化展示
- **快速导航**：提供常用功能模块的快捷入口，提升操作效率
- **业务概览**：实时展示关键业务指标，帮助客户快速了解业务状态
- **系统监控**：管理员可实时查看系统健康状态和数据治理任务
- **引导功能**：提供新手引导教程，帮助新客户快速上手

### 1.3 目标客户

- **管理员**：系统管理员、数据治理人员
- **业务客户**：运营人员、营销人员、业务分析师

## 4. 业务规则

### 4.1 时间段判断规则

#### 4.1.1 时间段计算
- **规则**：根据当前时间的小时数自动判断时间段
- **判断逻辑**：
  - 0-12点：显示"早上好"
  - 12-18点：显示"下午好"
  - 18-24点：显示"晚上好"
- **国际化**：支持简体、繁体、英文三种语言

### 4.2 角色化内容展示规则

#### 4.2.1 管理员看板
- **规则**：当客户角色为 `admin` 时，显示管理员专属内容
- **展示内容**：
  - 客户统计面板（4个统计卡片）
  - 系统状态监控（6个数据源系统状态）
  - 治理任务列表
  - 数据影响统计
  - 审计日志时间线
- **数据来源**：从 localStorage 读取客户角色信息

#### 4.2.2 业务客户看板
- **规则**：当客户角色为 `business` 或其他非管理员角色时，显示业务客户内容
- **展示内容**：
  - 业务指标卡片（4个指标）
  - 商机管道列表
  - 我的任务列表
- **数据来源**：从 localStorage 读取客户角色信息

### 4.3 快捷导航生成规则

#### 4.3.1 导航项过滤
- **规则**：根据客户角色和菜单权限动态生成快捷导航
- **管理员导航路径**：
  - `/dashboard/dataMonitor` - 数据监控看板
  - `/dataProcess/dataQualityWorkbench` - 数据质量工作台
  - `/dataProcess/errorCorrection` - 纠错处理
  - `/dataProcess/approval` - 审批中心
  - `/system/accountManage` - 账号管理
  - `/system/roleManage` - 角色管理
  - `/system/menuMange` - 菜单管理
  - `/system/operlogManage` - 操作日志
- **业务客户导航路径**：
  - `/customer/list` - 客户列表
  - `/customer/segmentation` - 客户分群
  - `/leadManagement/dashboard` - 线索看板
  - `/leadManagement/list` - 线索列表
  - `/tagManage` - 标签管理
  - `/segmentManage` - 分群管理
- **过滤规则**：
  - 只显示客户有权限访问的菜单
  - 隐藏 `isHide: true` 的菜单项
  - 按预定义路径顺序排列

### 4.4 系统状态判断规则

#### 4.4.1 系统状态汇总
- **规则**：根据所有数据源系统的状态计算总体状态
- **判断逻辑**：
  - 存在 `error` 或 `warning` 状态：显示"异常"标签（danger）
  - 存在 `gray`（待上传）状态：显示"未完成"标签（warning），显示待上传数量
  - 全部正常：显示"正常"标签（success）

#### 4.4.2 系统点击规则
- **规则**：只有 `manual` 类型且状态不为 `gray` 的系统可点击
- **点击行为**：跳转到数据质量工作台页面
- **上传按钮**：`gray` 状态的系统显示上传按钮，点击跳转到数据质量工作台

### 4.5 数据格式化规则

#### 4.5.1 数字格式化
- **规则**：大数字使用千分位分隔符格式化
- **示例**：`1342001` → `1,342,001`
- **实现**：使用 `toLocaleString('zh-CN')` 方法

## 5. 交互设计

### 5.1 页面布局

#### 5.1.1 头部区域
- **位置**：页面顶部
- **布局**：左侧欢迎信息 + 右侧操作按钮
- **内容**：
  - 时间段问候 + 客户名称
  - 欢迎消息
  - "开始导览"按钮
- **样式**：渐变背景、圆角卡片、动画效果

#### 5.1.2 统计面板区域（管理员）
- **位置**：头部下方
- **布局**：4个统计卡片横向排列（响应式网格）
- **卡片内容**：
  - 图标 + 数字 + 标签
  - 支持 hover 效果
- **响应式**：小屏幕自动换行

#### 5.1.3 快捷导航区域
- **位置**：统计面板下方
- **布局**：网格布局，自适应列数
- **卡片内容**：
  - 图标（带背景色）
  - 标题
  - 描述（可选）
- **交互**：点击跳转到对应页面，hover 显示阴影和位移效果

#### 5.1.4 主内容区域

##### 管理员看板布局
- **左侧（16列）**：
  - 系统状态卡片：6个数据源系统状态展示
  - 治理任务卡片：任务列表表格
- **右侧（8列）**：
  - 数据影响卡片：指标 + 图表
  - 审计日志卡片：时间线展示

##### 业务客户看板布局
- **顶部（24列）**：
  - 业务指标网格：4个指标卡片
- **下方**：
  - **左侧（16列）**：商机管道卡片
  - **右侧（8列）**：我的任务卡片

### 5.2 主要交互场景

#### 5.2.1 开始导览
- **触发方式**：点击头部"开始导览"按钮
- **交互流程**：
  1. 使用 driver.js 启动引导教程
  2. 根据客户角色显示不同的引导步骤
  3. 管理员：系统健康、治理任务、审计日志
  4. 业务客户：关键指标、智能商机
  5. 显示进度条，支持跳过

#### 5.2.2 快捷导航点击
- **触发方式**：点击快捷导航卡片
- **交互流程**：
  1. 检查路由路径是否存在
  2. 使用 `router.push()` 跳转
  3. 显示 hover 动画效果

#### 5.2.3 系统状态点击（管理员）
- **触发方式**：点击可点击的系统状态卡片
- **交互流程**：
  1. 检查系统类型是否为 `manual` 且状态不为 `gray`
  2. 跳转到数据质量工作台页面
  3. 显示提示信息（可选）

#### 5.2.4 数据上传（管理员）
- **触发方式**：点击 `gray` 状态系统的上传按钮
- **交互流程**：
  1. 跳转到数据质量工作台页面
  2. 显示提示："请在工作台上传数据"
  3. 按钮有脉冲动画效果

#### 5.2.5 查看治理任务详情
- **触发方式**：点击治理任务表格中的"修复"按钮
- **交互流程**：
  1. 跳转到对应的处理页面（根据任务类型）
  2. 或显示任务详情弹窗

#### 5.2.6 查看冲突（管理员）
- **触发方式**：点击数据影响卡片中的"查看冲突"按钮
- **交互流程**：
  1. 跳转到异常中心或纠错处理页面
  2. 显示冲突列表

#### 5.2.7 推送商机到BDC（业务客户）
- **触发方式**：点击商机列表中的"推送到BDC"按钮
- **交互流程**：
  1. 显示成功提示消息
  2. 调用后端API推送商机
  3. 更新商机状态（可选）

## 6. 异常处理

### 6.1 数据加载异常

#### 6.1.1 网络错误
- **场景**：获取客户信息或统计数据失败
- **处理**：
  - 显示错误提示："网络错误，请稍后重试"
  - 提供"重试"按钮
  - 记录错误日志
  - 使用默认数据或缓存数据（可选）

#### 6.1.2 角色信息读取失败
- **场景**：从 localStorage 读取角色信息失败
- **处理**：
  - 使用默认角色 `admin`
  - 控制台输出警告信息
  - 不影响页面正常显示

#### 6.1.3 菜单数据为空
- **场景**：客户没有可访问的菜单
- **处理**：
  - 隐藏快捷导航区域
  - 显示提示："暂无可用功能，请联系管理员"

### 6.2 数据展示异常

#### 6.2.1 统计数据为空
- **场景**：客户统计数据为 0 或 null
- **处理**：
  - 显示 `0` 或 `-`
  - 不显示错误提示
  - 保持卡片正常显示

#### 6.2.2 系统状态数据缺失
- **场景**：部分系统状态数据缺失
- **处理**：
  - 显示默认状态 `gray`
  - 显示"待上传"状态文本
  - 显示上传按钮
