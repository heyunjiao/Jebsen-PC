## 1. 产品概述

### 1.1 模块定位

异常中心是 C360 客户全景平台的数据质量保障核心模块，负责数据质量异常的检测、分类、处理和跟踪。支持异常分类（有效性、唯一性、完整性、一致性、合规性）、身份冲突解决、快速编辑、同步重试等功能。

### 1.2 业务价值

- **数据质量保障**：及时发现和处理数据质量问题
- **异常分类管理**：按异常类型分类处理，提高处理效率
- **冲突智能解决**：支持身份冲突的智能合并/拆分
- **处理流程化**：待处理/已处理状态管理，流程清晰

### 1.3 目标客户

- **数据质量专员**：处理数据异常、解决身份冲突
- **系统管理员**：监控数据质量、配置异常规则
- **运营人员**：查看和处理简单异常

### 3.2 异常分类枚举

#### 3.2.1 有效性异常 (validity)

| 异常类型 | 说明 | 示例 |
|---------|------|------|
| 格式错误 | 字段格式不符合规范 | 手机号格式错误 |
| 范围错误 | 字段值超出有效范围 | 年龄超过合理范围 |
| 类型错误 | 字段类型不匹配 | 数字字段包含字母 |

#### 3.2.2 唯一性异常 (uniqueness)

| 异常类型 | 说明 | 示例 |
|---------|------|------|
| 重复数据 | 同一字段值重复 | 同一手机号对应多个客户 |
| 身份冲突 | 身份信息冲突 | 同一身份证号对应多个客户 |

#### 3.2.3 完整性异常 (completeness)

| 异常类型 | 说明 | 示例 |
|---------|------|------|
| 必填字段缺失 | 必填字段为空 | 姓名字段为空 |
| 关键信息缺失 | 关键信息不完整 | 联系方式缺失 |

#### 3.2.4 一致性异常 (consistency)

| 异常类型 | 说明 | 示例 |
|---------|------|------|
| 数据不一致 | 不同字段数据不一致 | 性别与姓名不一致 |
| 跨源冲突 | 不同源系统数据冲突 | DMS和CRM数据不一致 |

#### 3.2.5 合规性异常 (compliance)

| 异常类型 | 说明 | 示例 |
|---------|------|------|
| 法律法规 | 违反法律法规 | 数据使用不符合GDPR |
| 业务规则 | 违反业务规则 | 不符合业务规范 |

## 4. 业务规则

### 4.1 异常检测规则

#### 4.1.1 自动检测触发
- **数据接入时**：新数据接入时自动检测
- **定时任务**：每天定时检测数据质量
- **数据更新时**：数据更新后触发检测

#### 4.1.2 异常分类规则
- **有效性检测**：格式、范围、类型验证
- **唯一性检测**：重复数据、身份冲突检测
- **完整性检测**：必填字段、关键信息检测
- **一致性检测**：数据一致性、跨源冲突检测
- **合规性检测**：法律法规、业务规则检测

### 4.2 异常处理规则

#### 4.2.1 处理模式分类
- **Mode A：身份冲突解决**
  - 触发条件：检测到身份冲突
  - 处理方式：合并/拆分工作台
  - 审批要求：高风险需审批
- **Mode B：数据质量快速编辑**
  - 触发条件：简单数据错误
  - 处理方式：快速编辑修复
  - 审批要求：低风险无需审批
- **Mode C：同步失败重试**
  - 触发条件：数据同步失败
  - 处理方式：重试同步
  - 审批要求：无需审批

#### 4.2.2 处理优先级
- **高优先级**：身份冲突、合规性异常
- **中优先级**：唯一性异常、一致性异常
- **低优先级**：有效性异常、完整性异常

### 4.3 身份冲突解决规则

#### 4.3.1 冲突识别
- **识别规则**：
  - 同一手机号对应多个客户客户
  - 同一证件号对应多个客户客户
  - 同一车牌号对应多个客户客户
- **置信度计算**：
  - 基于匹配字段数量
  - 基于数据来源可靠性
  - 基于历史合并记录

#### 4.3.2 合并规则
- **自动合并**：置信度 >= 95%，低风险
- **人工审批合并**：置信度 < 95%，高风险
- **合并策略**：
  - 保留主数据源数据
  - 合并多值字段
  - 统一OneID

#### 4.3.3 拆分规则
- **拆分条件**：确认是不同客户
- **拆分操作**：生成新的OneID
- **数据分配**：根据来源系统分配数据

### 4.4 批量操作规则

#### 4.4.1 批量忽略
- **适用场景**：确认无需处理的异常
- **操作限制**：仅限低风险异常
- **记录要求**：记录忽略原因

## 6. 异常处理

### 6.1 数据加载异常

#### 6.1.1 网络错误
- **场景**：网络请求失败
- **处理**：
  - 显示错误提示："网络错误，请稍后重试"
  - 提供"重试"按钮
  - 记录错误日志

#### 6.1.2 数据为空
- **场景**：查询结果为空
- **处理**：
  - 显示空状态提示："暂无异常数据"
  - 显示筛选条件说明

### 6.2 处理操作异常

#### 6.2.1 处理失败
- **场景**：异常处理失败
- **处理**：
  - 显示错误提示："处理失败：{错误信息}"
  - 保留处理表单数据
  - 允许重新提交

#### 6.2.2 审批被驳回
- **场景**：审批被驳回
- **处理**：
  - 显示驳回原因
  - 异常状态恢复为"待处理"
  - 允许重新处理

## 10. 接口定义

### 10.1 获取异常列表

**接口路径**：`POST /api/errorCorrection/list`

**请求参数**：
```typescript
{
  pageNum: number;
  pageSize: number;
  status?: "pending" | "processed";
  errorType?: string;
  // ... 其他筛选条件
}
```

**响应数据**：
```typescript
{
  code: number;
  msg: string;
  data: {
    list: ErrorTask[];
    total: number;
  };
}
```

### 10.2 获取统计面板数据

**接口路径**：`GET /api/errorCorrection/stats`

**响应数据**：
```typescript
{
  code: number;

  data: StatsPanel;
}
```

### 10.3 处理异常

**接口路径**：`POST /api/errorCorrection/process`

**请求参数**：
```typescript
{
  taskId: string;

  processResult: any;
  processComment?: string;
}
```

**接口路径**：`POST /api/errorCorrection/batchIgnore`

**请求参数**：
