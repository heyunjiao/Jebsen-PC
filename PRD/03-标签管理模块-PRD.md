## 1. 产品概述

### 1.1 模块定位

标签管理模块是 C360 客户全景平台的客户属性标记核心模块，支持自动化规则标签和人工手动标签的创建、管理和配置。提供规则编辑器、覆盖人数统计、审核流程等功能。

### 1.2 业务价值

- **客户属性标记**：通过标签快速识别客户特征
- **自动化打标**：基于规则自动为客户打标签，提高效率
- **标签分类管理**：按分类管理标签，便于查找和使用
- **覆盖统计**：实时统计标签覆盖人数和覆盖率

### 1.3 目标客户

- **运营人员**：创建和管理标签、配置标签规则
- **数据分析师**：使用标签进行客户分析
- **营销人员**：使用标签进行精准营销

## 4. 业务规则

### 4.1 标签创建规则

#### 4.1.1 标签名称规则
- **唯一性**：标签名称在同一分类下必须唯一
- **长度限制**：标签名称长度 2-50 个字符
- **字符限制**：支持中文、英文、数字、下划线

#### 4.1.2 规则配置规则
- **自动化标签**：必须配置规则
- **人工标签**：无需配置规则
- **规则验证**：保存前验证规则有效性

### 4.2 标签状态流转规则

#### 4.2.1 状态流转图
```
草稿(draft) → 已发布(published) → 已停用(disabled) → 已废弃(abandoned)
                ↓
            审核中(pending)
```

#### 4.2.2 状态转换规则
- **草稿 → 已发布**：需要审核（高风险规则）或直接发布（低风险规则）
- **已发布 → 已停用**：可随时停用
- **已停用 → 已发布**：可重新启用
- **已停用 → 已废弃**：确认不再使用后废弃

### 4.3 规则编辑器规则

#### 4.3.1 规则构建规则
- **支持逻辑**：AND、OR、NOT
- **支持操作符**：=、!=、>、>=、<、<=、in、not in、contains、not contains
- **支持字段类型**：文本、数字、日期、枚举、标签
- **规则深度**：支持多级嵌套（最多5层）

#### 4.3.2 规则验证规则
- **必填验证**：规则不能为空
- **格式验证**：验证规则格式正确性
- **逻辑验证**：验证规则逻辑合理性

### 4.4 覆盖统计规则

#### 4.4.1 统计计算规则
- **计算时机**：标签发布后自动计算，规则更新后重新计算
- **计算方式**：基于规则匹配客户数据
- **更新频率**：实时计算（T+0）

#### 4.4.2 覆盖率计算规则
- **覆盖率 = 覆盖人数 / 客户总数 × 100%**
- **客户总数**：系统中所有活跃客户数

### 4.5 审核流程规则

#### 4.5.1 审核触发规则
- **高风险规则**：置信度 < 95%，需要审核
- **低风险规则**：置信度 >= 95%，无需审核
- **审核人**：系统管理员或指定审核人

#### 4.5.2 审核结果规则
- **批准**：标签状态更新为"已发布"
- **驳回**：标签状态保持"草稿"，显示驳回原因

## 5. 交互设计

### 5.1 页面布局

#### 5.1.1 列表区域
- **位置**：页面主体
- **布局**：表格 + 操作按钮
- **功能**：
  - 标签列表展示
  - 搜索筛选
  - 状态筛选
  - 分类筛选

#### 5.1.2 操作按钮区域
- **位置**：表格上方
- **布局**：右侧操作按钮
- **功能**：
  - 新建标签
  - 批量操作（批量停用、批量废弃）

### 5.2 标签创建/编辑对话框

#### 5.2.1 对话框入口
- **触发方式**：点击"新建标签"按钮或"编辑"按钮
- **打开方式**：弹窗（Dialog）
- **弹窗宽度**：1000px

#### 5.2.2 表单布局
- **Tab 1：基本信息**
  - 标签名称
  - 标签分类
  - 标签类型
  - 描述
- **Tab 2：规则配置**（仅自动化标签）
  - 规则编辑器
  - 规则预览
  - 覆盖预估

#### 5.2.3 规则编辑器
- **布局**：树状结构展示
- **操作**：
  - 添加条件
  - 添加分组
  - 删除节点
  - 拖拽排序
- **字段选择**：下拉选择字段
- **操作符选择**：根据字段类型显示可用操作符
- **值输入**：根据字段类型显示对应输入组件

### 5.3 覆盖预估

#### 5.3.1 预估入口
- **触发方式**：规则配置后点击"覆盖预估"按钮
- **展示方式**：弹窗或侧边栏

#### 5.3.2 预估结果
- **预估人数**：显示预估覆盖人数
- **预估覆盖率**：显示预估覆盖率
- **样本数据**：显示匹配的样本客户（最多10条）
- **匹配原因**：显示每个样本的匹配原因

### 5.4 标签详情

#### 5.4.1 详情入口
- **触发方式**：点击标签名称或"详情"按钮
- **打开方式**：弹窗或侧边栏

#### 5.4.2 详情内容
- **基本信息**：标签名称、分类、类型、状态等
- **规则配置**：规则树状展示
- **覆盖统计**：覆盖人数、覆盖率
- **版本历史**：版本变更记录

## 6. 异常处理

### 6.1 数据加载异常

#### 6.1.1 网络错误
- **场景**：网络请求失败
- **处理**：显示错误提示，提供重试按钮

#### 6.1.2 数据为空
- **场景**：查询结果为空
- **处理**：显示空状态提示

### 6.2 规则配置异常

#### 6.2.1 规则验证失败
- **场景**：规则配置无效
- **处理**：显示验证错误信息，高亮错误节点

#### 6.2.2 覆盖预估失败
- **场景**：预估计算失败
- **处理**：显示错误提示，允许重新预估

## 10. 接口定义

### 10.1 获取标签列表

**接口路径**：`POST /api/tag/list`

**请求参数**：
```typescript
{
  pageNum: number;
  pageSize: number;
  tagName?: string;
  category?: TagCategory;
  tagType?: TagType;
  status?: TagStatus;
}
```

**响应数据**：
```typescript
{
  code: number;
  msg: string;
  data: {
    list: TagInfo[];
    total: number;
  };
}
```

### 10.2 创建/更新标签

**接口路径**：`POST /api/tag/add` 或 `PUT /api/tag/edit`

**请求参数**：
```typescript
{
  tagId?: string;
  tagName: string;
  category: TagCategory;
  tagType: TagType;
  description: string;
  ruleConfig?: RuleConfig;
}
```

### 10.3 模拟验证规则

**接口路径**：`POST /api/tag/simulate`

**请求参数**：
```typescript
{
  ruleConfig: RuleConfig | RuleNode;
}
```

**响应数据**：
```typescript
{
  code: number;
  msg: string;
  data: {
    estimatedCount: number;
    estimatedRate: string;
    sampleData: Array<{
      oneId: string;
      name: string;
      matchReason: string;
    }>;
  };
}
```
